# SCUM服务器查询插件

SCUM游戏服务器状态查询插件，支持绑定服务器ID、搜索服务器、卡密授权管理等功能。

## 功能特性

- 支持带 `/` 和不带 `/` 前缀的命令
- 服务器绑定功能，绑定后一键查询
- 简化格式显示，查看更直观
- 卡密授权系统，支持分群授权和时间控制
- 白名单群组功能
- 全局主人账号管理
- **群组自定义设置**：每个群组可以单独配置"允许查询其他服务器"
- **全局默认设置**：未自定义设置的群组使用全局配置
- **克隆设置功能**：快速复制一个群组的设置到另一个群组

## 指令列表

### 普通命令

| 指令 | 说明 | 示例 |
|------|------|------|
| `id查询` | 查询已绑定的服务器状态（简化格式） | `id查询` |
| `id查询 <关键词>` | 搜索SCUM服务器 | `id查询 中国` |
| `id查询 <8位以上数字>` | 查询指定ID的服务器详细信息 | `id查询 38615532` |
| `id查询 search <关键词>` | 搜索SCUM服务器 | `id查询 search China` |
| `查询在线` | 查询已绑定服务器的简化状态（推荐） | `查询在线` |
| `查看设置` | 查看当前群组的设置 | `查看设置` |
| `scum帮助` | 显示完整帮助信息 | `scum帮助` |

### 管理命令（仅管理员/群主/主人）

| 指令 | 说明 | 示例 |
|------|------|------|
| `绑定id <服务器ID>` | 绑定服务器ID到当前群组 | `绑定id 38615532` |
| `解绑` | 解除当前群组的服务器ID绑定 | `解绑` |
| `设置 允许查询其他服务器 <true/false>` | 配置是否允许查询其他服务器 | `设置 允许查询其他服务器 true` |
| `设置 重置` | 重置群组设置为全局默认 | `设置 重置` |

### 授权命令

| 指令 | 说明 | 示例 |
|------|------|------|
| `激活卡密 <卡密>` | 激活卡密授权当前群组 | `激活卡密 JK301234567890ABC` |
| `授权查询` | 查询当前群组授权状态 | `授权查询` |

### 主人命令

| 指令 | 说明 | 示例 |
|------|------|------|
| `生成卡密 <天数> [群组ID]` | 生成指定天数的卡密 | `生成卡密 30` |
| `群组授权 <群组ID> <天数>` | 直接为指定群组授权 | `群组授权 123456789 30` |
| `克隆设置 <源群组ID> [目标群组ID]` | 复制一个群组的设置到另一个群组 | `克隆设置 123456789 987654321` |

## 使用说明

### 1. 基础查询（白名单群组）

白名单群组可以直接使用所有查询功能：

```
id查询 中国
id查询 38615532
```

### 2. 绑定服务器（管理员/群主/主人）

首次使用需要绑定服务器：

```
绑定id 38615532
```

### 3. 查询在线（绑定后）

绑定后，直接使用 `查询在线` 查看服务器状态：

```
查询在线
```

显示格式：
```
【服务器名称】
人数: 77/128
时间: 15:05
状态: 在线
区服: 中国
IP: 202.189.14.25:7052
```

### 4. 群组设置

#### 查看当前设置

```
查看设置
```

显示：
```
📋 当前群组设置：
  允许查询其他服务器: True

当前设置来源: 群组自定义
```

#### 修改设置

管理员或群主可以修改当前群组的设置：

```
设置 允许查询其他服务器 false
```

可用参数：
- `true`, `1`, `是`, `启用`, `开启` - 启用
- `false`, `0`, `否`, `禁用`, `关闭` - 禁用

#### 重置为全局默认

```
设置 重置
```

### 5. 克隆设置（仅主人）

主人可以复制一个群组的设置到另一个群组：

```
克隆设置 123456789 987654321
```

如果在目标群组中发送命令，可以省略目标群组ID：

```
克隆设置 123456789
```

### 6. 授权系统

#### 启用授权

在插件设置中开启 `启用授权系统`，非白名单群组需要卡密授权才能使用。

#### 生成卡密（主人操作）

```
生成卡密 30
```

生成30天有效期的卡密。

如需限定特定群组使用：
```
生成卡密 30 123456789
```

#### 激活卡密（管理员操作）

群管理员在群组中激活卡密：

```
激活卡密 JK301234567890ABCDEF12
```

#### 查询授权状态

```
授权查询
```

显示当前群组的授权到期时间和剩余天数。

## 插件配置

| 配置项 | 说明 | 默认值 |
|--------|------|--------|
| 白名单群组 | 不需要绑定ID和授权即可使用的群组ID列表 | [] |
| 管理员群组 | 需要管理员权限才能使用绑定/解绑功能的群组ID列表（留空则所有群组可用） | [] |
| 默认启用 | 未在白名单中的群组是否需要绑定ID才能使用查询功能 | true |
| 全局设置:允许查询其他服务器 | 绑定ID后是否允许查询其他服务器（全局默认设置） | true |
| 主人账号 | 全局主人账号ID列表（主人可以无视管理员/群主要求使用管理命令） | [] |
| 主人完全无限制 | 主人是否不受授权和绑定限制 | true |
| 启用授权系统 | 是否启用卡密授权系统 | false |
| 管理密钥 | 用于生成和验证卡密的密钥（建议修改为复杂密钥） | joker_plugin_secret_key_2024 |

## 权限说明

- **主人账号**：可以生成卡密、使用绑定/解绑等所有管理命令，支持克隆设置
- **管理员/群主**：可以使用绑定/解绑、修改群组设置
- **普通成员**：可以使用查询命令、查看设置
- **白名单群组**：无需绑定和授权即可使用所有功能
- **授权群组**：需要激活卡密后才能使用（非白名单群组）

## 设置说明

### 全局 vs 群组设置

- **全局设置**：在插件配置中设置，作为所有群组的默认值
- **群组设置**：每个群组可以单独覆盖全局设置
- **设置来源**：使用 `查看设置` 可以查看当前使用的是群组自定义还是全局默认

### 允许查询其他服务器

- 当群组已绑定服务器ID时，此项控制是否允许查询其他服务器
- 启用后，绑定群组的成员仍然可以使用 `id查询` 搜索和查询其他服务器
- 禁用后，绑定群组只能查询已绑定的服务器

### 数据存储

- 绑定信息存储在 `group_bindings.json`
- 授权数据存储在 `auth_data.json`
- 群组设置存储在 `group_settings.json`

## 注意事项

1. 所有命令均支持带 `/` 和不带 `/` 两种方式
2. 卡密格式为 `JK + 天数 + 时间戳 + 签名`，请勿修改
3. 主人账号在配置中添加，格式为纯数字ID
4. 建议修改默认的管理密钥以提高安全性
5. 群组设置克隆仅复制自定义设置，不复制绑定关系
